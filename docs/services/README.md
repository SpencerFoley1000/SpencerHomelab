# Services

This directory documents homelab services.

Each service should have its own page once it becomes part of the environment or a meaningful experiment.

## Service Documentation Standard

Use [`templates/service-template.md`](templates/service-template.md) when adding a new service.

At minimum, document:

- Purpose
- Host
- Deployment method
- Dependencies
- Networking
- Storage
- Backup strategy
- Recovery procedure
- Security considerations
- Maintenance notes

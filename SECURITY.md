# Security and Public Documentation Policy

This repository is public and intentionally sanitized.

## Scope

The repository documents architecture, design decisions, operational practices, troubleshooting, and lessons learned from a personal homelab. It does not expose sensitive operational details required to access or attack the environment.

## Information Not Published

The following information must not be committed:

- Passwords, API keys, tokens, private keys, recovery codes, cookies, or secrets.
- Public IP addresses or ISP account information.
- Exact personally identifying SSIDs or home network names.
- Home address details or family-identifying information.
- Device serial numbers, asset tags, license keys, or private certificates.
- Full internal layouts where exact values are not necessary to explain the design.

## Sanitized Placeholders

Use placeholders where specific values are unnecessary or sensitive:

- `<LAN_SUBNET>`
- `<MGMT_NETWORK>`
- `<HOST_IP>`
- `<PRIVATE_DNS>`
- `<REDACTED_SSID>`
- `<SECRET_STORED_IN_PASSWORD_MANAGER>`

## Disclosure Note

This is a personal learning and portfolio repository. If a security issue is found in the documentation itself, open a GitHub issue without including sensitive exploit details.
